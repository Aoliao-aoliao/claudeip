Claude Code 域名分流规则大全

确保 Claude Code 的每一个请求都走代理，一个域名都不漏。包含核心域名、CDN、认证、监控、NTP 时区检测的完整规则。

为什么需要这份规则？Claude Code 在运行时不仅仅访问 claude.ai 和 api.anthropic.com，它还会请求 CDN 资源、认证服务、监控上报、甚至 NTP 时间同步。任何一个域名漏掉没走代理，都可能导致 DNS 泄露、地域不一致、甚至触发风控。

NTP 时区检测 — 容易被忽略的关键规则

为什么分流规则里要加 NTP？Claude Code 会通过 UDP 协议访问 NTP 时间服务器来判断你的时区。如果 NTP 请求直连了本地网络，返回的时区信息会和你代理出口的时区不一致。这是一个很隐蔽的风控触发点——你的 API 请求说自己在海外，但系统时间却是中国时区。
规则用途类型
geosite:category-ntp所有已知 NTP 时间服务器NTP

这条规则确保所有 NTP 时间同步请求也走代理出口，返回的时区信息和你的代理地区一致。配合 禁用 IPv6，可以彻底消除时区泄露的风险。

注意：NTP 走的是 UDP 协议，很多代理客户端默认不处理 UDP。确保你的配置开启了 UDP 代理，或者用 WebRTC 泄露检测 验证 UDP 出口是否正常。

如何验证规则是否生效

配好规则之后，建议用以下工具逐一验证：

  1. Claude AI IP 检测 — 查看 Claude 出口 IP 是否走了你指定的节点，以及 IP 的信任评分
  2. IP 查询 + 分流测试 — 在分流测试表格里找到 claude.ai 和 anthropic.com，确认它们走的是同一个出口
  3. DNS 泄露检测 — 确保 DNS 查询没有泄露到本地运营商
  4. WebRTC 泄露检测 — 确保 UDP 流量（包括 NTP）没有绕过代理
  5. 全球 Ping 测试 — 测试你选择的代理出口到各地区的延迟，选最优路径
这份规则会随着 Anthropic 的服务变化而更新。如果你发现有新的域名没有被覆盖，欢迎在使用 Claude AI IP 检测 时留意是否有异常出口。
